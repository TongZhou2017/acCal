# 订阅功能使用指南

本文档说明如何使用 acCal 的 RSS 订阅和邮件订阅功能。

## 📡 RSS 订阅

### 什么是 RSS？

RSS (Really Simple Syndication) 是一种内容聚合格式，允许您通过 RSS 阅读器订阅网站更新，无需频繁访问网站。

### 如何订阅 RSS？

1. **获取 RSS 地址**
   - 访问网站的订阅页面：`/subscribe/`
   - RSS Feed 地址：`{{ site.url }}{{ site.baseurl }}/feed.xml`

2. **使用 RSS 阅读器**
   - **在线服务**：Feedly, Inoreader, The Old Reader
   - **浏览器扩展**：RSS Subscription Extension (Chrome)
   - **桌面应用**：NetNewsWire (Mac), FeedReader (Windows/Linux)
   - **移动应用**：Feedly, NewsBlur

3. **添加订阅**
   - 复制 RSS 地址
   - 在 RSS 阅读器中点击"添加订阅"或"Add Feed"
   - 粘贴 RSS 地址
   - 完成！

### RSS 订阅的优势

- ✅ 实时更新：新会议添加后立即推送
- ✅ 隐私保护：无需提供邮箱地址
- ✅ 统一管理：在一个地方查看所有订阅
- ✅ 离线阅读：支持下载内容离线查看

## 📧 邮件订阅

### 订阅流程

1. **访问订阅页面**
   - 点击网站导航栏的"订阅"链接
   - 或直接访问：`/subscribe/`

2. **填写订阅信息**
   - **邮箱地址**：输入您的邮箱（必填）
   - **学科领域**：选择您感兴趣的学科（至少选择一个）
     - 生态学 (Ecology)
     - 进化生物学 (Evolution)
     - 环境科学 (Environment)
     - 生物信息学 (Bioinformatics)
   - **默认提醒天数**：设置默认的提前提醒天数（默认30天）

3. **设置特定会议提醒**（可选）
   - 点击"添加会议提醒"
   - 从列表中选择会议
   - 设置该会议的提前提醒天数

4. **提交订阅**
   - 点击"提交订阅"按钮
   - 系统会创建一个 GitHub Issue 来处理您的订阅请求
   - 我们将在24小时内处理并发送确认邮件

### 邮件类型

订阅后，您将收到以下类型的邮件：

1. **订阅确认邮件**
   - 订阅成功后立即发送
   - 包含您的订阅信息确认

2. **新会议通知**
   - 当有新会议添加到您订阅的学科领域时发送
   - 包含会议基本信息和快速操作链接

3. **会议提醒**
   - 在会议开始前按您设置的天数发送
   - 包含会议详细信息和重要日期

4. **截稿提醒**
   - 在摘要截稿日期前7天发送
   - 提醒您及时提交摘要

### 管理订阅

#### 修改订阅设置

1. 访问订阅页面：`/subscribe/`
2. 重新填写表单并提交
3. 系统会更新您的订阅设置

#### 取消订阅

- 回复任何订阅邮件中的取消链接
- 或访问网站取消订阅页面（如已配置）

#### 为特定会议设置提醒

**方法一：在订阅页面设置**
1. 访问订阅页面
2. 在"为特定会议设置提醒"部分添加会议
3. 设置提前提醒天数

**方法二：在会议详情页设置**
1. 打开任意会议详情页
2. 点击"设置提醒"按钮
3. 填写邮箱和提醒天数
4. 提交

### 提醒天数说明

- **默认提醒天数**：适用于所有订阅的会议
- **特定会议提醒天数**：仅适用于该会议，优先级高于默认设置
- **截稿提醒**：固定为截稿日期前7天，不可自定义

### 隐私保护

- ✅ 我们不会分享您的邮箱地址给第三方
- ✅ 所有订阅数据仅用于发送会议信息
- ✅ 您可以随时取消订阅
- ✅ 订阅请求通过 GitHub Issues 处理，过程透明

## 🔧 技术实现

### RSS Feed

- 使用 Jekyll 的 `jekyll-feed` 插件自动生成
- Feed 地址：`/feed.xml`
- 包含所有已发布的会议信息
- 符合 RSS 2.0 标准

### 邮件订阅

当前实现方式：
- 通过 GitHub Issues 收集订阅请求
- 使用邮件模板生成邮件内容
- 需要配置邮件发送服务（SendGrid, Mailgun 等）

未来计划：
- 集成 GitHub Actions 自动化处理
- 支持邮件服务 API 集成
- 提供订阅管理后台

## 📝 邮件模板

邮件模板位于 `_email_templates/` 目录：

- `subscription_confirmation.html` - 订阅确认邮件
- `conference_reminder.html` - 会议提醒邮件
- `new_conference_notification.html` - 新会议通知
- `deadline_reminder.html` - 截稿提醒邮件

所有模板支持变量替换，详见各模板文件中的注释。

## ❓ 常见问题

### Q: RSS Feed 无法访问？
A: 确保 Jekyll 构建成功，jekyll-feed 插件已正确安装。访问 `/feed.xml` 检查是否正常生成。

### Q: 没有收到确认邮件？
A: 
1. 检查垃圾邮件文件夹
2. 确认邮箱地址填写正确
3. 等待24小时（处理时间）
4. 检查 GitHub Issue 是否已创建

### Q: 如何修改提醒天数？
A: 重新访问订阅页面，修改设置并提交。系统会更新您的订阅配置。

### Q: 可以订阅多个学科吗？
A: 可以！在订阅表单中勾选多个学科领域即可。

### Q: RSS 和邮件订阅有什么区别？
A: 
- **RSS**：需要 RSS 阅读器，实时更新，隐私友好
- **邮件**：直接发送到邮箱，支持个性化提醒，需要提供邮箱
